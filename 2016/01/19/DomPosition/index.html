clientHeight和clientWidth属性

clientHeight和clientWidth返回的是content+padding的宽度

该属性指的是元素的可视部分宽度和高度，即content+padding，如果没有滚动条，即为元素设定的高度和宽度，如果出现滚动条，滚动条会遮盖元素的宽高，那么该属性就是其本来宽高减去滚动条的宽高

document元素的clientHeight和clientWidth属性，就代表了网页的大小。

　　function getViewport(){
　　　　if (document.compatMode == "BackCompat"){
　　　　　　return {
　　　　　　　　width: document.body.clientWidth,
　　　　　　　　height: document.body.clientHeight
　　　　　　}
　　　　} else {
　　　　　　return {
　　　　　　　　width: document.documentElement.clientWidth,
　　　　　　　　height: document.documentElement.clientHeight
　　　　　　}
　　　　}
　　}
上面的getViewport函数就可以返回浏览器窗口的高和宽。

1）这个函数必须在页面加载完成后才能运行，否则document对象还没生成，浏览器会报错。

2）大多数情况下，都是document.documentElement.clientWidth返回正确值。但是，在IE6的quirks模式中，document.body.clientWidth返回正确的值，因此函数中加入了对文档模式的判断。

3）clientWidth和clientHeight都是只读属性，不能对它们赋值。

clientTop和clientLeft属性

clientTop和clientLeft返回的是border的宽度

offsetHeight和 offsetWidth属性

content+padding+border的宽度

offsetLeft和 offsetTop属性

offsetLeft和 offsetTop返回的是该元素border边缘相对于已定位的offsetParent对象左上角的距离

offsetParent是指包裹该元素的父级元素，父级元素需定位，如无offsetParent，则相对的是文档的左上角

scrollWidth和scrollHeight属性

scrollWidth和scrollHeight返回的是该元素content+padding加上溢出尺寸(溢出尺寸即超出包裹div尺寸，滚动条以外的尺寸）

上面属性都不可写

scrollLeft和scrollTop属性

scrollLeft和scrollTop指的是当元素其中的内容超出其宽高的时候，元素被滚动卷起的高度和宽度。

此属性可写，改变后自动滚动到该位置

相对坐标和绝对坐标

相对坐标是指到浏览器窗口左上角的距离，绝对坐标是指到文档左上角的距离。

getBoundingClientRect()方法是获取该元素的相对坐标，后加top和
left属性可得到相应坐标

绝对坐标的计算有两种方法

（1）可以根据offsetLeft和 offsetTop属性获取元素的绝对定位，即元素到文档左上角的距离，将该元素的offsetLeft和所有父级元素的offsetLeft累加可得，函数如下

/**
 * 获取元素绝对定位left
 *
 * @param  {Object} element 元素节点
 * @return {Number}  actualLeft 元素节点相对于文档的绝对left位置
 */
function getElementLeft(element){
    var actualLeft = element.offsetLeft;
    var current = element.offsetParent;
    while (current !== null){
        actualLeft += current.offsetLeft;
        current = current.offsetParent;
    }
    return actualLeft;
}

（2）可以用相对坐标加文档卷起的宽度得到绝对定位

function y(e){
    alert(this.getBoundingClientRect()+document.documentElement.scrollTop)
}

//如果document.documentElement.scrollTop不起作用，始终为0，则将其改为document.body.scrollTop

相对坐标的计算也有两种方法

（1）直接用getBoundingClientRect()方法可得

（2）用绝对坐标减去卷起宽度

function z(e){
    alert(getElementTop(this)-document.body.scrollTop)
}

Event对象

clientX和clientY

这对属性是当事件发生时，鼠标点击位置相对于浏览器（可视区）的坐标，即浏览器左上角坐标的（0,0），该属性以浏览器左上角坐标为原点，计算鼠标点击位置距离其左上角的位置.不管浏览器窗口大小如何变化，都不会影响点击位置的坐标。

offsetX和offsetY

这一对属性是指当事件发生时，鼠标点击位置相对于该事件源的位置，即点击该div，以该div左上角为原点来计算鼠标点击位置的坐标

screenX和screenY

screenX和screenY是事件发生时鼠标相对于屏幕的坐标
点击浏览器视窗内的同一点，浏览器在屏幕里改变位置，该点位置就会改变

pageX和pageY

该属性是事件发生时鼠标点击位置相对于页面的位置，通常浏览器窗口没有出现滚动条时，该属性和event.clientX及event.clientY是等价的，但是当浏览器出现滚动条的时候，pageX通常会大于clientX，因为页面还存在被卷起来的部分的宽度和高度
