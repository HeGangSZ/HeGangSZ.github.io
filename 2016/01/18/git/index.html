git学习记录

Git是目前世界上最先进的分布式版本控制系统

生成新文件夹，用命令行进入该文件夹

生成新的仓库

$ git init

//生成新的仓库，会生成一个.git文件夹，用于版本控制

添加文件到暂存区

$ git add <file>

$ git add -A  //添加所有文件

//添加文件到暂存区

提交文件

$ git commit -m "description"

//""里面是对提交的描述，提交会将暂存区的文件全部提交上去

没有加入暂存区的更改不会被提交，所以每次commit都要先add

查看提交状态

$ git status

所有文件都已被提交，没有更改

//所有文件都已被提交，没有更改
On branch master
nothing to commit, working directory clean

test.test修改了但是没提交

//test.test修改了但是没提交
On branch master
Changes not staged for commit:
  (use "git add <file>..." to update what will be committed)
  (use "git checkout -- <file>..." to discard changes in working directory)

        modified:   test.txt

no changes added to commit (use "git add" and/or "git commit -a")

two.txt没有add到repository

//two.txt没有add到repository
Untracked files:
  (use "git add <file>..." to include in what will be committed)

        two.txt

查看修改内容

$ git diff

查看历史记录

$ git log

//结果从上到下是从新到旧的提交版本
//后加--pretty=oneline参数简化输出内容
//commit后面一串是版本号，可用于版本回退
//gitlog太长用q退出

查看命令历史

$ git reflog
//如果已经回退到之前版本无法知道新版本的版本号，可通过这条命令查看

版本回退

//版本回退会将repository和工作区的版本都回退
//版本回退会使暂存区和工作区的版本同时回退

$ git reset --hard HEAD^
//HEAD表示当前版本
//回退到上一个版本（现版本的倒数第二个版本）

$ git reset --hard HEAD^^
//回退到现版本的倒数第三个版本

$ git reset --hard HEAD~100
//回退到现版本的倒数第101个版本

$ git reset --hard 3628164
//回退到版本号前面数字为3628164的版本，版本号查看git log可见

$ git reset HEAD readme.txt    
//已将修改add到暂存区，此命令撤销暂存区的修改

撤销修改

$ git checkout -- readme.txt

//如果readme.txt未add到暂存区，则工作区撤销修改到最新版本的内容
//如果readme.txt已add到暂存区，则工作区撤销修改到添加到暂存区后的状态
//如果工作区的文件误删，可以用此命令恢复到最新版本

$ git reset HEAD readme.txt

//已将修改add到暂存区，此命令撤销暂存区的修改

本地修改未add到暂存区，则用git checkout – test.txt

本地修改add到暂存区了，git checkout – test.txt恢复的是暂存区的状态

add到暂存区但没提交，暂存区状态可用git reset HEAD test.txt恢复到上一次提交的版本库的状态

所以add后想恢复则先用git reset HEAD test.txt将暂存区状态恢复到版本库状态，再用git checkout – test.txt 将工作区状态恢复到暂存区状态：版本库——暂存区——工作区，恢复顺序和提交顺序相反

删除文件

$ rm readme.txt     //从工作区删除
$ git rm -- readme.txt  //从暂存区删除
$ git commit -m "rmReadMe"   //从版本库里删除

//从版本库中删去readme.txt
//同理
（1）如果本地误删但未add到暂存区，可用git checkout -- readme.txt从暂存区恢复到本地；
（2）如果add到暂存区了，先git reset HEAD test.txt再git checkout -- test.txt；
（3）如果最后提交了，则先git reset --hard HEAD^回退版本再进行第二步操作

创建分支

//创建dev分支，然后切换到dev分支：

$ git checkout -b dev
Switched to a new branch 'dev'

//git checkout命令加上-b参数表示创建并切换，相当于以下两条命令：
$ git branch dev  //创建分支
$ git checkout dev  //切换分支
Switched to branch 'dev'

查看分支

$ git branch
//查看分支

合并分支

$ git merge dev
//合并dev分支到当前分支（必须切换到需要合并的主分支将次分支合并金主分支里）

$ git merge --no-ff dev
//使用--no-ff参数可以禁用Fast forward模式来合并分支，这样后续可以查看分支合并的信息

$ git log --graph --pretty=oneline --abbrev-commit
//用这个命令可以看到分支合并的信息

删除分支

$ git branch -d <name>
删除分支

$ git branch -D <name>
强行删除一个没有合并过的分支

保存状态

$ git stash
//保存未提交的工作状态

$ git stash pop   //恢复后删除stash
$ git stash apply  //恢复后不删除stash，需用git stash drop删除
//恢复工作状态 

$ git stash list
//查看保存工作状态的位置



$ git stash apply stash@{0}
//恢复指定工作状态

//当分支中的修改和另一分支有冲突时，没有提交是不能checkout到另一分支的，如果此时有需要创建新分支修改bug，则可以用$ git status命令来保存状态，再使用git stash pop恢复
//也可多次git stash这样就有多个保存位置，这时用git stash apply stash@{0}恢复指定工作状态

创建标签

$ git tag v1.0
//给某版本创建版本标签

git show <tagname>
//显示标签名

$ git tag
//查看所有标签

push an existing repository from the command line

git remote add origin https://github.com/HeGangSZ/learngit.git
git push -u origin master
//将内容提交到master分支，提交到其他分支要改master为其他分支名，以后提交到github输入git push origin master


//提交前要先添加SSH
//添加SSH

(1)git输入

    $ ssh-keygen -t rsa -C "邮件地址@youremail.com"

期间会要求输入password，输入字符不会显示出来，如果输入则每次上传时都会要求输入，不输入则按回车继续

(2)添加SSH Key到GitHub

把生成的id_rsa.pub里信息复制到在github的setting里的ssh中

到个人邮箱确认github发的确认邮件

(3)ssh连接测试

    git输入 $ ssh -T git@github.com

(4)设置用户信息

git输入

    $ git config --global user.name "用户名"

    $ git config --global user.email  "邮件地址@youremail.com"

成功信息如下：

Hi cnfeat! You've successfully authenticated, but GitHub does not provide shell access.
